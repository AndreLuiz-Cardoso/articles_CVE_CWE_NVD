Calculating CVSS Scores — A Practical Example

Understanding CVSS scoring in theory is one thing. Seeing how it applies to a real vulnerability scenario makes it concrete and actionable.

The Scenario

A remote code execution vulnerability exists in a widely used web server software. The vulnerability allows an attacker to execute arbitrary code on the server remotely, without requiring any authentication or user interaction.

Identifying the CVSS Base Metrics

CVSS v3.1 uses eight base metrics to calculate a score. Let's assess each for this scenario:

Attack Vector (AV): Network
The vulnerability is exploitable over the internet. Any attacker with network access to the web server can attempt the exploit. This receives the highest-risk value: Network (N).

Attack Complexity (AC): Low
The attack does not require special conditions, race conditions, or complex configurations. Any attacker can attempt it straightforwardly. Value: Low (L).

Privileges Required (PR): None
No authentication is required. The attacker does not need a user account or any form of credentials. Value: None (N).

User Interaction (UI): None
The vulnerability can be exploited without any action from a legitimate user, no clicking links, no opening files. Value: None (N).

Scope (S): Changed
Successful exploitation allows the attacker to affect components beyond the vulnerable web server itself, for example, accessing the underlying operating system or other services. Value: Changed (C).

Confidentiality Impact (C): High
The attacker can read any data on the system, including sensitive configuration files, credentials, and user data. Value: High (H).

Integrity Impact (I): High
The attacker can modify or delete any files on the system, including web application code. Value: High (H).

Availability Impact (A): High
The attacker can crash the web server or consume all system resources, making it unavailable. Value: High (H).

Calculating the CVSS Score

Using the CVSS v3.1 calculator with these values:

AV:N / AC:L / PR:N / UI:N / S:C / C:H / I:H / A:H

CVSS Base Score: 10.0 — CRITICAL

This is the maximum possible CVSS score. The scenario represents a worst-case vulnerability: remotely exploitable, no authentication required, no user interaction needed, and capable of completely compromising the target system.

Implications and Mitigation

A CVSS score of 10.0 demands immediate emergency response:

Immediate actions (within hours):


Take the vulnerable web server offline or isolate it from the internet if possible
Apply the vendor's emergency patch immediately, no waiting for change windows
If no patch is available, implement temporary mitigations: firewall rules blocking access to the vulnerable component, WAF rules filtering malicious requests


Short-term actions (within days):


Audit all systems running the vulnerable software
Review logs for signs of exploitation (unusual process creation, unexpected outbound connections)
Implement network segmentation to limit what an attacker could access from a compromised web server


Long-term actions:


Establish processes to receive and act on vendor security advisories
Implement runtime application self-protection (RASP) to detect exploitation attempts
Conduct a post-incident review to understand why the vulnerability existed and was not caught earlier


A 10.0 CVSS score is rare but not unheard of. Log4Shell (CVE-2021-44228) received exactly this score, and the response it demanded, patching hundreds of millions of systems worldwide, reflects the severity that score represents.
