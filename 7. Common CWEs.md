Common CWEs and Their Real-World Impact

Understanding which CWEs are most prevalent and most dangerous helps development teams focus their security efforts where they will have the greatest impact.

CWE-89: SQL Injection

What it is: The application includes user-supplied data in database queries without proper sanitization, allowing attackers to manipulate the query structure.

Real-world impact: SQL injection has enabled some of the largest data breaches in history. The 2008 Heartland Payment Systems breach (134 million credit cards stolen) used SQL injection as the initial entry point. SQL injection in a product search bar gives an attacker a path to every record in the database.

Priority: Critical. Any application using a database should be tested for SQL injection before deployment and after any change to database query code.

CWE-79: Cross-Site Scripting (XSS)

What it is: The application includes user-supplied data in web pages without proper encoding, allowing attackers to inject malicious JavaScript that executes in other users' browsers.

Real-world impact: XSS enables session hijacking (stealing authentication cookies), credential theft (fake login forms), and malware distribution. The 2005 Samy worm spread to one million MySpace profiles in under 24 hours using a stored XSS vulnerability.

Priority: High. XSS vulnerabilities are extremely common (appearing in millions of web applications) and the consequences for users can be severe.

CWE-20: Improper Input Validation

What it is: The application fails to verify that input conforms to expected format, type, length, and range before processing it.

Real-world impact: Improper input validation is a root cause for many other weaknesses. Buffer overflows (CWE-120), format string vulnerabilities (CWE-134), and many injection attacks all stem from insufficient input validation.

Priority: High. Input validation should be implemented at every system boundary — API endpoints, file upload handlers, form fields, and command-line parameters.

CWE-287: Improper Authentication

What it is: Authentication mechanisms can be bypassed or circumvented, allowing unauthorized access.

Real-world impact: Weak or bypassable authentication is the direct cause of unauthorized account access. This includes weak password requirements, missing authentication on API endpoints, and authentication logic errors.

Priority: Critical for any system handling user accounts or sensitive data.

Prioritization Framework

When prioritizing CWE remediation in a development project, consider three factors: how common the weakness is in the existing codebase (scan results), how severe the impact would be if exploited (CVSS scores of related CVEs), and how easy it is to exploit (exploit complexity). Rank your remediation backlog by the product of these three factors to address the highest-risk weaknesses first.
